# 小红书竞品分析（2026.03 采集）

## 头部账号

### 1. 星宿叭叭
- **定位**：星宿IP人格化，漫画风
- **代表作**："28星宿平行宇宙｜在古代从事什么工作"（1940赞）、"1分钟入门星宿"（3817赞）
- **特点**：
  - 把28个星宿做成角色IP，给每个星宿设定古代身份
  - 漫画风格，17张图讲清楚一个知识框架
  - 评论区非常活跃，用户自发代入角色
  - **很强的版权意识**，公开声明不允许抄袭洗稿
- **启示**：人格化+漫画是验证过的方向，但不能模仿她的具体角色设定

### 2. 讲星宿的小凡
- **定位**：星宿关系排行/合盘分析
- **代表作**："最受欢迎的星宿关系排行"（1910赞）
- **特点**：
  - 插画风，用原创卡通人物
  - 评论区核心行为：用户报自己和对象的关系类型，互相对照
  - 评论区非常适合做"查宿/合盘"互动
- **启示**：排行榜型内容天然引发讨论和@朋友

### 3. 闲窗小记
- **定位**：传统文化科普
- **代表作**："古人的二十八星宿，更适合中国人体质的星座"（1.2万赞）
- **特点**：
  - 用清代丁冠鹏《五星二十八宿册》做配图
  - 最后一张图是生日对照表，引爆收藏
  - 标题用"更适合中国人体质的星座"做反差钩子
- **启示**：生日对照表是刚需，收藏率极高

## 评论区行为模式

### 高频互动类型

1. **查宿请求** (最多)
   - "我X月X日的是什么宿？"
   - "980721阳历，本命和值日星宿分别是什么"

2. **合盘查询** (第二多)
   - "X宿和Y宿怎么样？"
   - "奎宿和柳宿怎么样"
   - "柳宿跟亢宿如何"

3. **关系类型讨论** (第三多)
   - "中友衰千万不要入局，要死了"
   - "近荣亲体验感没得说"
   - "远安坏明明一整个恨海情天"
   - "业胎有什么好谈的"

4. **代入角色**
   - "所以哀家是垂帘听政呢还是上梁山呢"
   - "我是昴，对象张"
   - "好好好我和男友一个客栈老板一个酒馆掌柜"

5. **质疑/追问**
   - "这是自创的还是古代本来就记载了的"
   - "看值日还是本命"
   - "有没有对应十二星座的"

6. **学术讨论**
   - 星宿命名规律（七曜+动物）
   - 与西方星座的对比
   - 历史典籍来源

## 爆款公式总结

1. **工具型**：生日对照表 / 关系自查表 → 高收藏
2. **排行型**：最受欢迎关系排行 / 最虐关系排行 → 高评论
3. **人格化**：星宿角色 + 古代身份 → 高代入感
4. **反差型**："更适合中国人体质的星座" → 高传播
5. **合盘型**：两人关系分析 → 高@转发

## 差异化机会

现有竞品的空白：
- **AI生图风格**：目前都是手绘/漫画，还没有AI绘图风格的星宿内容
- **合盘工具化**：没有做到"评论区输入生日，自动回复关系"的自动化
- **视觉统一的系列**：大多数号的视觉不够统一，没有形成品牌识别
- **中英双语**：没有人做 Pinterest 版本
