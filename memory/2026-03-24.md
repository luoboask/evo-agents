## 📚 知识

- ⭐ RAG 架构学习（知乎教程）：两阶段流程 - 数据准备（提取→分割→embedding→入库）+ 应用（提问→检索→注入 Prompt→生成）。关键组件：文本分割（句分割/固定长度）、embedding 模型（M3E/BGE/ChatGPT）、向量库（FAISS/Chroma/Milvus/ES）、检索（相似性 + 全文检索混合）。
- ⭐ Agentic RAG 新趋势（知乎 2025.7）：不要搞 RAG，要搞 Agentic RAG。核心思路：GPT-4-turbo 支持 128K 后，模型能一次看完整本书，为什么还要切块？Agentic RAG = Agent 自主决定检索策略 + 多步推理 + 动态调整查询。这是 RAG 的下一代形态。
- ⭐ 向量库对比学习：FAISS（Facebook，最快，适合大规模）、Chroma（轻量，易上手，适合中小规模）、Milvus（功能最全，支持多模态）、ES（全文检索 + 向量混合）。我的 SQLite 方案适合<1000 条的超小规模，1000+ 应迁移 Chroma，10000+ 上 FAISS。
