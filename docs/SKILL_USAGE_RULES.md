# 技能使用规则

_什么时候该用什么技能？这里是明确的行为指南。_

---

## 🧠 记忆相关技能

### memory-search（混合记忆系统）

**何时使用：**
| 场景 | 是否使用 | 说明 |
|------|---------|------|
| 用户提到"之前说过..." | ✅ 必须 | 搜索历史对话 |
| 用户提到"记住这个" | ✅ 必须 | 记录到记忆系统 |
| 开始新会话时 | ✅ 必须 | 读取昨天/今天的记忆文件 |
| 回答需要上下文的问题 | ✅ 必须 | 先搜索记忆再回答 |
| 日常闲聊 | ❌ 不需要 | 无需搜索 |
| 简单事实查询 | ❌ 不需要 | 直接用知识回答 |

**使用方式：**
```python
# 记录
memory.record_interaction("user", "用户说的话")

# 搜索
results = memory.search("查询内容", context="medium")
```

**触发关键词：**
- "之前说过"
- "记得吗"
- "上次"
- "记住"
- "我之前提到"

### rag（RAG 评估系统）

**何时使用：**
| 场景 | 是否使用 | 说明 |
|------|---------|------|
| 使用记忆搜索后 | ✅ 建议 | 记录检索质量 |
| 每周例行检查 | ✅ 必须 | 生成性能报告 |
| 用户反馈检索不准 | ✅ 必须 | 分析问题并调优 |
| 新部署 RAG 系统 | ✅ 必须 | 建立基线指标 |
| 日常对话 | ❌ 不需要 | 无需评估 |

**使用方式：**
```python
# 评估
evaluate(days=7, report=True)

# 自动调优
auto_tune()

# 获取指标
get_metrics(days=7)
```

**触发时机：**
- 每周日自动执行
- 用户反馈"搜索不准"时
- 系统性能下降时

## 🧬 自进化技能

### self-evolution（自进化系统 v5.0）

**何时使用：**
| 场景 | 是否使用 | 说明 |
|------|---------|------|
| 完成重要任务后 | ✅ 必须 | 记录 TASK_COMPLETED |
| 学到新知识后 | ✅ 必须 | 记录 KNOWLEDGE_GAINED |
| 解决复杂问题后 | ✅ 建议 | 执行分形思考 |
| 每天结束时 | ✅ 必须 | 执行夜间循环 |
| 收到用户反馈后 | ✅ 必须 | 记录 FEEDBACK_RECEIVED |
| 技能改进后 | ✅ 必须 | 记录 SKILL_IMPROVED |
| 日常对话 | ❌ 不需要 | 无需记录 |

**使用方式：**
```python
# 记录事件
evolve(type="TASK_COMPLETED", content="完成 memory-search 技能优化")

# 分形思考
fractal(limit=20)

# 夜间循环
nightly()

# 查看状态
status()
```

**触发时机：**
- 任务完成 → 立即记录
- 每天凌晨 2 点 → 自动夜间循环
- 每周日 → 分形思考

## 🌐 知识获取技能

### web-knowledge（网页搜索）

**何时使用：**
| 场景 | 是否使用 | 说明 |
|------|---------|------|
| 用户问实时信息 | ✅ 必须 | 新闻、天气、股价等 |
| 用户问你不知道的事 | ✅ 必须 | 先搜索再回答 |
| 需要验证信息 | ✅ 必须 | 交叉验证 |
| 已知知识范围内 | ❌ 不需要 | 直接用已有知识 |
| 数学计算 | ❌ 不需要 | 直接计算 |

**使用方式：**
```bash
python3 skills/web-knowledge/search.py "查询内容"
```

**触发关键词：**
- "最新"
- "今天"
- "现在"
- "最近"
- "2026 年"

## ⚠️ 常见错误

### ❌ 错误：不搜索记忆就回答
```
用户："我之前说过我喜欢吃什么"
错误回答："我不知道"
正确做法：先调用 memory-search 搜索
```

### ❌ 错误：完成任务不记录
```
完成技能开发后不记录
后果：自进化系统没有数据，无法成长
正确做法：evolve(type="TASK_COMPLETED", ...)
```

### ❌ 错误：从不评估 RAG 性能
```
后果：检索质量下降但不知道
正确做法：每周 evaluate(report=True)
```

### ❌ 错误：夜间循环不执行
```
后果：记忆没有整合，知识没有结构化
正确做法：配置定时任务每天执行 nightly()
```

## 🎯 最佳实践

### 会话开始时
1. 读取 `memory/YYYY-MM-DD.md`（今天 + 昨天）
2. 如果在主会话，读取 `MEMORY.md`
3. 如果是当天第一次会话，运行每日回顾

### 会话进行中
1. 用户提到历史 → 搜索记忆
2. 用户问实时信息 → 搜索网页
3. 重要事件发生 → 记录到自进化系统

### 会话结束时
1. 记录本次会话到记忆系统
2. 如果有重要学习 → 记录 KNOWLEDGE_GAINED
3. 如果是当天最后一次 → 执行夜间循环

### 定期任务
- **每天**：夜间循环（凌晨 2 点）
- **每周**：RAG 评估报告（周日）
- **每周**：分形思考（周日）
- **每月**：记忆整理和清理
