# 🎯 自我进化触发机制

_定义何时、如何触发自我进化_

## 触发器类型

### 1️⃣ 即时触发 (Immediate Triggers)

**触发时机**: 每次交互后立即检查

| 触发条件 | 阈值 | 优先级 | 行动 |
|---------|------|--------|------|
| 高失败率 | > 30% | 🔴 Critical | 分析失败并修复 |
| 响应过慢 | > 30秒 | 🟠 High | 优化性能 |
| 重复错误 | 同一错误3次 | 🟠 High | 修复根因 |

**示例场景**:
```
用户: "搜索 XXX"
→ 搜索失败（超时）
→ 检查: 失败率 50% > 30%
→ 🔴 触发: analyze_failures_and_fix
→ 自动创建 search_v2 并添加重试机制
```

### 2️⃣ 周期性触发 (Periodic Triggers)

**触发时机**: 按固定时间周期

| 周期 | 时间 | 优先级 | 行动 |
|-----|------|--------|------|
| 每日反思 | 23:00 | 🟡 Medium | 日终反思报告 |
| 每周复盘 | 周日 00:00 | 🟡 Medium | 周回顾和规划 |
| 每月进化 | 每月1号 | 🟠 High | 月度进化计划 |

**示例场景**:
```
时间: 2026-03-16 23:00
→ 🟡 触发: daily_retrospective
→ 生成今日反思报告
→ 识别改进机会
→ 更新知识图谱
```

### 3️⃣ 阈值触发 (Threshold Triggers)

**触发时机**: 资源使用超过阈值

| 资源 | 阈值 | 优先级 | 行动 |
|-----|------|--------|------|
| 记忆大小 | > 50 MB | 🟡 Medium | 归档旧记忆 |
| 技能数量 | > 20 个 | 🟢 Low | 合并相似技能 |
| 交互次数 | > 100/天 | 🟢 Low | 优化工作流 |

**示例场景**:
```
记忆大小: 52 MB > 50 MB
→ 🟡 触发: archive_old_memories
→ 归档 30 天前的记忆
→ 压缩历史数据
→ 更新 MEMORY.md
```

### 4️⃣ 用户触发 (User Initiated)

**触发时机**: 用户主动请求

| 触发方式 | 优先级 | 行动 |
|---------|--------|------|
| 输入 "evolve" | 🟠 High | 完整进化检查 |
| 满意度 < 0.5 | 🟠 High | 针对性改进 |

**示例场景**:
```
用户: "evolve"
→ 🟠 触发: full_evolution_check
→ 运行完整健康检查
→ 生成改进计划
→ 执行高优先级改进
```

## 触发流程

```
┌─────────────────────────────────────────────────────────┐
│                    交互/事件/时间                        │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│              触发器检查 (EvolutionTriggers)              │
├─────────────────────────────────────────────────────────┤
│  1. 即时检查 → 失败率? 响应时间? 重复错误?              │
│  2. 周期检查 → 时间到了吗?                              │
│  3. 阈值检查 → 资源超限了吗?                            │
│  4. 用户检查 → 用户请求了吗?                            │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │   发现触发器?           │
              └────────────────────────┘
                    │是          │否
                    ▼            ▼
        ┌──────────────────┐   ┌─────────────┐
        │ 按优先级排序      │   │ 继续监控    │
        │ 执行对应行动      │   │             │
        └──────────────────┘   └─────────────┘
                    │
                    ▼
        ┌──────────────────────────┐
        │      进化执行            │
        │  - 分析根因              │
        │  - 生成改进方案          │
        │  - 实施改进              │
        │  - 验证效果              │
        │  - 记录到知识图谱        │
        └──────────────────────────┘
```

## 配置文件

```json
{
  "immediate_triggers": {
    "failure_rate_threshold": 0.3,
    "response_time_threshold": 30,
    "error_pattern_threshold": 3
  },
  "periodic_triggers": {
    "daily_reflection": "23:00",
    "weekly_review": "Sunday",
    "monthly_evolution": 1
  },
  "threshold_triggers": {
    "memory_size_mb": 50,
    "skill_count": 20,
    "interaction_count": 100
  },
  "user_initiated": {
    "command": "evolve",
    "feedback_threshold": 0.5
  }
}
```

## 使用方法

```bash
# 检查所有触发器
python3 skills/self-reflection/triggers.py

# 查看配置
python3 skills/self-reflection/triggers.py --config

# 模拟触发
python3 skills/self-reflection/triggers.py --simulate failure
python3 skills/self-reflection/triggers.py --simulate slow
```
